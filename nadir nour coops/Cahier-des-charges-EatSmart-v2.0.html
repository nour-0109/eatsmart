Instructions pour le livrable EatSmart v2.0

Démo du résultat final

Cahier des charges

1. Navigation

Vous devez créer un menu de navigation comprenant deux liens :

  1. Menu : Ce lien pointe vers la page de menu de votre restaurant.
  2. Contact : Ce lien pointe vers une page contact contenant un formulaire.

2. Structure de la page

Chaque page doit être structurée en plusieurs blocs :

  1. Header (en-tête de page)
  2. Footer (pied de page)

Pour cela :

  • Créez un fichier header.php contenant le code du header.
  • Créez un fichier footer.php contenant le code du footer.

Ces fichiers seront injectés dans chaque page du site à l’aide des commandes PHP suivantes :

<?php require_once(__DIR__ . '/header.php'); ?>
<?php require_once(__DIR__ . '/footer.php'); ?>

À faire : Recherchez sur internet comment utiliser ces commandes si nécessaire.

3. Données dynamiques

Les données du menu (nom des plats, images, descriptions, prix) doivent être :

  • Stockées dans un fichier variables.php.
  • Injectées dans les pages où elles sont nécessaires.

4. Page Contact

La page contact doit contenir un formulaire avec les éléments suivants :

  1. Un champ email pour que l’utilisateur saisisse son adresse email.
  2. Une zone de texte permettant d’écrire un message sur plusieurs lignes.
  3. Un bouton Submit permettant de soumettre le formulaire.

5. Page de confirmation

Une fois que l’utilisateur appuie sur le bouton Submit, il est redirigé vers une page intitulée submit_contact.php.
Sur cette page, les données envoyées par le formulaire doivent être affichées, notamment :

  1. L’adresse email saisie.
  2. Le contenu du message.

Les données du formulaire doivent être stockées dans l’URL.
Par exemple, l’URL après soumission pourrait ressembler à ceci :
submit_contact.php?email=utilisateur@example.com&message=Bonjour

À faire : Recherchez sur internet comment transmettre et récupérer des données via l’URL.

Organisation sur GitHub

  1. Ajoutez ces fichiers dans votre repository GitHub existant sous le dossier Eatsmart v2.0.
  2. Assurez-vous que le dossier contient au minimum les fichiers suivants :
    • header.php
    • footer.php
    • variables.php
    • contact.php
    • submit_contact.php

Délais

Vous devez publier les fichiers de la version EatSmart v2.0 sur votre repository GitHub au plus tard jeudi prochain.
Le repository doit être accessible pour évaluation.

Critères de notation

Voici les critères sur lesquels je vais évaluer votre travail :

  1. Respect des consignes et complétude du cahier des charges.
  2. Fonctionnalité et propreté du code PHP et HTML.
  3. Structure et organisation du repository GitHub.
  4. Originalité et présentation visuelle de votre site.

Bonne chance et faites preuve de créativité ! 🎉
